How it works

Three steps to a self-building project

01

Describe your project

Answer a short set of questions: what you're building, who it's for, what must ship in version 1, and your deadline. No technical knowledge required.

02

Connect GitHub

Paste a Personal Access Token. We create a repo on your account with all agents pre-configured — the memory system, the roadmap, the architecture file. Nothing for you to write.

03

Activate two routines

In Claude Code, set up two routines that take two minutes to configure. One runs every Monday to plan the week. One runs every weekday to implement and review. Then step back.

Why it works

Everything managed for you

Runs every day without you

Routines fire on a schedule. A manager agent plans. A coder agent implements. A reviewer validates and merges. You check in when you want to.

No setup beyond two clicks

The repo, the agent configs, the memory files, the CI pipeline — all generated for you. You don't write a single config file.

Keeps a memory of your project

Agents read and write to structured files in your repo — goals, architecture, lessons learned. Context carries over between sessions automatically.

Works with your Claude plan

Uses Claude Code routines built into Claude.ai. Pro, Max, Team and Enterprise plans all supported. No extra API keys needed to start.

Before you start

What you need

A GitHub account

Free tier works. You'll need to generate a Personal Access Token with repo scope.

A Claude account with routines access

Claude Pro or above. Routines are available on Pro (5/day), Max (15/day), Team and Enterprise plans.
